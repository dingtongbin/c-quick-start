# 第5章 字符串

## 5.1 字符串基本操作

### 字符串在C语言中的表示与基础操作

在C语言中，字符串是以空字符'\0'结尾的字符数组。这是C语言处理文本数据的基础方式，虽然看起来简单，但却是所有高级字符串操作的根基。

| 功能名称       | 函数调用           | 说明                               |
| -------------- | ------------------ | ---------------------------------- |
| 字符串长度获取 | strlen(str)        | 返回字符串中字符个数（不包括'\0'） |
| 字符串复制     | strcpy(dest, src)  | 将src复制到dest，包括'\0'          |
| 字符串连接     | strcat(dest, src)  | 将src连接到dest末尾                |
| 字符串比较     | strcmp(str1, str2) | 比较两个字符串，返回值表示大小关系 |

下面是一个简单的字符串操作示例：

```c
#include <stdio.h>
#include <string.h>

int main() {
    // 定义字符串，注意要预留足够的空间
    char str1[20] = "Hello";
    char str2[] = "World";
    char result[50];
    
    // 获取字符串长度
    printf("str1长度: %zu\n", strlen(str1));  // 输出5
    
    // 复制字符串
    strcpy(result, str1);  // 将str1复制到result
    printf("复制后: %s\n", result);
    
    // 连接字符串
    strcat(result, " ");   // 先加空格
    strcat(result, str2);  // 再连接str2
    printf("连接后: %s\n", result);
    
    // 比较字符串
    int cmp = strcmp(str1, str2);
    if (cmp == 0) {
        printf("两个字符串相等\n");
    } else if (cmp < 0) {
        printf("str1字典序小于str2\n");
    } else {
        printf("str1字典序大于str2\n");
    }
    
    return 0;
}
```

需要注意的是，使用这些标准库函数时必须包含`<string.h>`头文件，并且要确保目标字符串有足够的空间来存储操作结果，否则会导致缓冲区溢出，这是一个常见的安全漏洞。

字符串的基本操作是后续所有字符串算法的基础，掌握这些操作对于理解更复杂的字符串处理算法至关重要。

## 5.2 朴素模式匹配

### 朴素模式匹配算法原理

朴素模式匹配（也称为暴力匹配）是最直观的字符串匹配算法。它的基本思想是从主串的每一个位置开始，逐个字符与模式串进行比较，直到找到完全匹配的位置或者遍历完整个主串。

算法的时间复杂度为O(n*m)，其中n是主串长度，m是模式串长度。虽然效率不高，但实现简单，易于理解。

| 功能名称 | 算法步骤             | 说明                                                         |
| -------- | -------------------- | ------------------------------------------------------------ |
| 初始化   | 设置主串和模式串索引 | i=0为主串索引，j=0为模式串索引                               |
| 匹配过程 | 逐字符比较           | 如果字符相等，两个索引都前进；如果不等，主串索引回退到上次开始位置的下一个，模式串索引重置为0 |
| 结果返回 | 返回匹配位置或-1     | 如果找到匹配，返回起始位置；否则返回-1                       |

下面是朴素模式匹配的C语言实现：

```c
#include <stdio.h>
#include <string.h>

// 朴素模式匹配函数
// 参数: 主串s，模式串p
// 返回: 第一次匹配的位置，未找到返回-1
int naiveStringMatch(const char* s, const char* p) {
    // 获取主串和模式串长度
    int sLen = strlen(s);
    int pLen = strlen(p);
    
    // 边界检查：如果模式串为空，返回0
    if (pLen == 0) return 0;
    // 如果主串长度小于模式串，不可能匹配
    if (sLen < pLen) return -1;
    
    // 从主串的每个可能位置开始尝试匹配
    for (int i = 0; i <= sLen - pLen; i++) {
        int j;
        // 在当前位置i开始，逐个比较字符
        for (j = 0; j < pLen; j++) {
            // 如果发现不匹配的字符，跳出内层循环
            if (s[i + j] != p[j]) {
                break;
            }
        }
        // 如果j等于模式串长度，说明完全匹配
        if (j == pLen) {
            return i;  // 返回匹配的起始位置
        }
    }
    
    // 遍历完所有可能位置都没有找到匹配
    return -1;
}

int main() {
    char text[] = "ABABCABABA";
    char pattern[] = "ABABA";
    
    int pos = naiveStringMatch(text, pattern);
    
    if (pos != -1) {
        printf("模式串在位置 %d 找到匹配\n", pos);
        // 验证结果
        printf("匹配的子串: %.*s\n", (int)strlen(pattern), text + pos);
    } else {
        printf("未找到匹配\n");
    }
    
    return 0;
}
```

这个实现中，外层循环控制主串的起始位置，内层循环进行实际的字符比较。当内层循环完成且j等于模式串长度时，说明找到了完全匹配。

朴素模式匹配算法虽然简单直观，但在最坏情况下（例如主串为"AAAAA"，模式串为"AAAAB"）需要进行大量的比较操作，效率较低。这也是为什么我们需要更高效的KMP算法。

## 5.3 KMP算法

### KMP算法核心思想

KMP（Knuth-Morris-Pratt）算法是一种高效的字符串匹配算法，它通过预处理模式串来避免在匹配失败时回退主串指针，从而将时间复杂度优化到O(n+m)。

KMP算法的核心是构建"部分匹配表"（也称为next数组），这个表记录了模式串中每个位置的最长相同前后缀长度。利用这个信息，当匹配失败时，我们可以直接跳转到模式串的适当位置继续匹配，而不需要回退主串指针。

| 功能名称     | 算法组件             | 说明                                                   |
| ------------ | -------------------- | ------------------------------------------------------ |
| next数组构建 | 计算模式串的前缀函数 | 对于每个位置i，计算pattern[0..i]的最长相同真前后缀长度 |
| 主匹配过程   | 利用next数组进行匹配 | 当字符不匹配时，根据next数组调整模式串位置             |
| 时间复杂度   | O(n+m)               | n为主串长度，m为模式串长度                             |

首先，我们需要构建next数组：

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

// 构建KMP算法的next数组
// 参数: 模式串pattern，模式串长度len
// 返回: next数组指针
int* buildNextArray(const char* pattern, int len) {
    // 分配next数组内存
    int* next = (int*)malloc(len * sizeof(int));
    if (next == NULL) {
        printf("内存分配失败\n");
        return NULL;
    }
    
    // next[0]总是0，因为单个字符没有真前后缀
    next[0] = 0;
    
    // len为模式串长度，j表示当前要计算的next值的索引
    // i表示当前正在比较的前缀位置
    int i = 0;  // 前缀指针
    int j = 1;  // 后缀指针
    
    // 从第二个字符开始计算next值
    while (j < len) {
        if (pattern[i] == pattern[j]) {
            // 如果前缀和后缀字符相等
            next[j] = i + 1;  // 最长相同前后缀长度为i+1
            i++;  // 前缀指针前进
            j++;  // 后缀指针前进
        } else {
            // 如果不相等
            if (i == 0) {
                // 如果前缀指针已经在开头，说明没有相同前后缀
                next[j] = 0;
                j++;  // 只移动后缀指针
            } else {
                // 回退前缀指针到next[i-1]的位置
                i = next[i - 1];
            }
        }
    }
    
    return next;
}
```

接下来是KMP主匹配函数：

```c
// KMP字符串匹配主函数
// 参数: 主串text，模式串pattern
// 返回: 第一次匹配的位置，未找到返回-1
int kmpSearch(const char* text, const char* pattern) {
    int textLen = strlen(text);
    int patternLen = strlen(pattern);
    
    // 边界情况处理
    if (patternLen == 0) return 0;
    if (textLen < patternLen) return -1;
    
    // 构建next数组
    int* next = buildNextArray(pattern, patternLen);
    if (next == NULL) return -1;
    
    int i = 0;  // 主串指针
    int j = 0;  // 模式串指针
    
    // 开始匹配过程
    while (i < textLen) {
        if (text[i] == pattern[j]) {
            // 字符匹配，两个指针都前进
            i++;
            j++;
            
            // 如果模式串完全匹配
            if (j == patternLen) {
                int result = i - j;  // 计算匹配位置
                
                // 释放next数组内存
                free(next);
                return result;
            }
        } else {
            // 字符不匹配
            if (j == 0) {
                // 如果模式串指针在开头，只能移动主串指针
                i++;
            } else {
                // 根据next数组跳转模式串指针
                j = next[j - 1];
            }
        }
    }
    
    // 释放next数组内存
    free(next);
    return -1;  // 未找到匹配
}

// 测试函数
int main() {
    char text[] = "ABABCABABA";
    char pattern[] = "ABABA";
    
    int pos = kmpSearch(text, pattern);
    
    if (pos != -1) {
        printf("KMP算法在位置 %d 找到匹配\n", pos);
        printf("匹配的子串: %.*s\n", (int)strlen(pattern), text + pos);
    } else {
        printf("未找到匹配\n");
    }
    
    return 0;
}
```

KMP算法的关键在于next数组的构建和使用。next数组告诉我们当匹配失败时，模式串应该跳转到哪个位置继续匹配，这样就避免了主串指针的回退，大大提高了匹配效率。

虽然KMP算法的实现比朴素匹配复杂，但它在线性时间内完成匹配的能力使其在处理大量文本数据时具有显著优势。理解KMP算法不仅有助于解决字符串匹配问题，还能加深对算法优化思想的认识。
