# 第6章 树的基础

## 6.1 树的基本概念

树是一种非线性的数据结构，它由节点（或称为结点）组成，这些节点通过边连接形成层次关系。在计算机科学中，树被广泛应用于文件系统、数据库索引、编译器设计等领域。

### 树的定义

树是由n（n≥0）个有限节点组成的一个具有层次关系的集合。当n=0时，称为空树。在任意一棵非空树中：

1. 有且仅有一个特定的称为根（Root）的节点
2. 当n>1时，其余节点可分为m（m>0）个互不相交的有限集合T₁, T₂, ..., Tₘ，其中每一个集合本身又是一棵树，并且称为根的子树（SubTree）

### 树的基本术语

- **节点**：树中的每个元素称为节点
- **度**：节点拥有的子树数目称为节点的度
- **叶子节点**：度为0的节点称为叶子节点或终端节点
- **分支节点**：度不为0的节点称为分支节点或非终端节点
- **父节点**：若一个节点含有子节点，则这个节点称为其子节点的父节点
- **子节点**：一个节点含有的子树的根节点称为该节点的子节点
- **兄弟节点**：具有相同父节点的节点互称为兄弟节点
- **层次**：从根开始定义起，根为第1层，根的子节点为第2层，以此类推
- **深度**：树中节点的最大层次称为树的深度或高度
- **森林**：由m（m≥0）棵互不相交的树的集合称为森林

### 树的特点

1. 树中的每个节点都有且仅有一个直接前驱（除了根节点）
2. 树中所有节点都可以从根节点到达
3. 树中不存在环路
4. 具有n个节点的树必定有n-1条边

树这种数据结构就像我们现实生活中的家谱一样，从祖先开始，一代一代往下延伸，每个祖先可以有多个后代，但每个后代只能有一个直接的父亲。这种层次分明的结构让我们能够高效地组织和查找数据。

## 6.2 树的存储结构

在计算机中表示树结构有多种方法，每种方法都有其适用的场景和优缺点。下面我们介绍几种常见的树存储结构。

### 双亲表示法

双亲表示法采用一组连续的存储空间来存储树的各个节点，同时在每个节点中附设一个指示器，指示其双亲节点在数组中的位置。

```c
#define MAX_TREE_SIZE 100

// 树节点结构定义
typedef struct {
    int data;           // 节点数据
    int parent;         // 双亲位置，-1表示根节点
} PTNode;

// 树结构定义
typedef struct {
    PT.Node nodes[MAX_TREE_SIZE];  // 节点数组
    int n;              // 节点数量
} PTree;
```

双亲表示法的优点是寻找双亲节点非常方便，时间复杂度为O(1)。但缺点是寻找子节点比较困难，需要遍历整个数组。

### 孩子表示法

孩子表示法将每个节点的孩子节点排列起来，以单链表作为存储结构，再用一个数组存储所有链表的头指针。

```c
#define MAX_TREE_SIZE 100

// 孩子节点结构
typedef struct CTNode {
    int child;          // 孩子节点在数组中的位置
    struct CTNode *next; // 指向下一个孩子节点
} CTNode;

// 节点结构
typedef struct {
    int data;           // 节点数据
    CTNode *firstchild; // 第一个孩子节点指针
} CTBox;

// 树结构
typedef struct {
    CTBox nodes[MAX_TREE_SIZE]; // 节点数组
    int n;              // 节点数量
    int r;              // 根节点位置
} CTree;
```

孩子表示法便于实现与孩子有关的操作，比如遍历某个节点的所有子节点。但对于查找双亲节点则不太方便。

### 孩子兄弟表示法（二叉链表表示法）

孩子兄弟表示法又称二叉树表示法，以二叉链表作为树的存储结构。每个节点包含三个域：数据域、第一个孩子指针域和右兄弟指针域。

```c
// 树节点结构定义
typedef struct CSNode {
    int data;                   // 节点数据
    struct CSNode *firstchild;  // 指向第一个孩子节点
    struct CSNode *nextsibling; // 指向右兄弟节点
} CSNode, *CSTree;
```

孩子兄弟表示法的最大优点是可以将复杂的树结构转化为二叉树结构，从而利用二叉树的各种算法来处理一般的树。这种方法既便于查找孩子节点，也便于查找兄弟节点。

### 树存储结构的选择

选择哪种存储结构主要取决于具体的应用需求：

- 如果需要频繁查找节点的双亲，应选用双亲表示法
- 如果需要频繁查找节点的孩子，应选用孩子表示法
- 如果需要同时访问节点的孩子和兄弟，或者希望将树转换为二叉树处理，应选用孩子兄弟表示法

树的存储结构选择就像是选择合适的工具箱，不同的工具适合不同的工作。理解每种存储结构的特点，能帮助我们在实际编程中做出最合适的选择。

## 6.3 二叉树的定义与性质

二叉树是树结构中最重要、应用最广泛的一种特殊形式。它不仅是许多高级数据结构的基础，也是理解更复杂树形结构的关键。

### 二叉树的定义

二叉树是n（n≥0）个节点的有限集合，它或者是空集（n=0），或者由一个根节点及两棵互不相交的分别称作这个根的左子树和右子树的二叉树组成。

从定义可以看出，二叉树具有以下特点：

1. 二叉树可以为空，即没有节点
2. 二叉树的每个节点最多有两个子树，分别称为左子树和右子树
3. 左子树和右子树是有顺序的，不能随意交换位置
4. 即使某节点只有一棵子树，也要区分是左子树还是右子树

### 二叉树的基本形态

1. **空二叉树**：没有任何节点的二叉树
2. **只有一个根节点的二叉树**
3. **只有左子树的二叉树**
4. **只有右子树的二叉树**
5. **完全二叉树**：除最后一层外，其他层都被完全填满，且最后一层的节点都集中在左侧
6. **满二叉树**：所有层都被完全填满的二叉树

### 二叉树的重要性质

**性质1**：在二叉树的第i层上至多有2^(i-1)个节点（i≥1）

**性质2**：深度为k的二叉树至多有2^k - 1个节点（k≥1）

**性质3**：对任何一棵二叉树T，如果其叶子节点数为n₀，度为2的节点数为n₂，则n₀ = n₂ + 1

**性质4**：具有n个节点的完全二叉树的深度为⌊log₂n⌋ + 1

**性质5**：对于完全二叉树，如果按照从上到下、从左到右的顺序对节点进行编号（从1开始），则对于任意节点i（1≤i≤n）：

- 如果i=1，则节点i是根节点，无双亲
- 如果i>1，则节点i的双亲节点为⌊i/2⌋
- 如果2i≤n，则节点i的左孩子为2i；否则无左孩子
- 如果2i+1≤n，则节点i的右孩子为2i+1；否则无右孩子

### 特殊二叉树类型

**满二叉树**：深度为k且有2^k - 1个节点的二叉树称为满二叉树。满二叉树的特点是每一层的节点数都达到最大值。

**完全二叉树**：深度为k的具有n个节点的二叉树，当且仅当其每一个节点都与深度为k的满二叉树中编号从1至n的节点一一对应时，称为完全二叉树。

**平衡二叉树**：左右子树的高度差不超过1的二叉树。

**二叉搜索树**：对于树中任意节点，其左子树中所有节点的值都小于该节点的值，右子树中所有节点的值都大于该节点的值。

二叉树就像是一棵精心修剪过的树，每个分支最多只有两个方向，这样的结构既简洁又高效。掌握二叉树的定义和性质，为我们后续学习各种树的操作和应用打下了坚实的基础。
