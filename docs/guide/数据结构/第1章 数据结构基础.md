### 1.1 数据结构概述

在编程世界里，数据结构就像是厨师的刀具——不同的食材需要不同的刀法。数据结构就是我们处理数据的"刀法"，它决定了我们如何组织、存储和操作数据。

**实例方法表格**

| 功能名称   | 实例调用方法                                    | 具体功能与注意事项                   |
| ---------- | ----------------------------------------------- | ------------------------------------ |
| 定义结构体 | `struct Node { int data; struct Node* next; };` | 创建自定义数据类型，用于组织相关数据 |
| 内存分配   | `malloc(sizeof(struct Node))`                   | 动态分配内存，需检查返回值是否为NULL |
| 内存释放   | `free(ptr)`                                     | 释放动态分配的内存，避免内存泄漏     |

**使用示例**

```c
#include <stdio.h>    // 包含标准输入输出库
#include <stdlib.h>   // 包含标准库函数，如malloc和free

// 定义一个简单的节点结构体，包含数据和指向下一个节点的指针
struct Node {
    int data;                    // 存储整型数据
    struct Node* next;           // 指向下一个相同类型节点的指针
};

int main() {
    // 动态分配内存创建新节点，sizeof计算结构体大小
    struct Node* newNode = (struct Node*)malloc(sizeof(struct Node));
    
    // 检查内存分配是否成功，防止程序崩溃
    if (newNode == NULL) {
        printf("内存分配失败！\n");  // 输出错误信息
        return -1;                   // 返回错误码
    }
    
    // 为节点的数据成员赋值
    newNode->data = 42;              // 设置节点存储的数据为42
    
    // 打印节点数据验证操作成功
    printf("节点数据: %d\n", newNode->data);  // 输出: 节点数据: 42
    
    // 释放之前分配的内存，防止内存泄漏
    free(newNode);
    newNode = NULL;                  // 将指针置空，避免悬空指针
    
    return 0;                        // 程序正常结束
}
```

**注意事项**

- 所有动态分配的内存都必须手动释放，否则会造成内存泄漏
- 使用指针前必须确保其指向有效内存地址
- 结构体定义通常放在头文件中以便复用

数据结构是程序设计的基础，它帮助我们以高效的方式组织数据。掌握基本的数据结构概念是学习更复杂算法的前提。

### 1.2 算法与复杂度分析

如果说数据结构是刀具，那么算法就是菜谱。好的算法能让程序运行得更快、更省资源。复杂度分析就是评估这个"菜谱"效率的方法。

**实例方法表格**

| 功能名称       | 实例调用方法     | 具体功能与注意事项                            |
| -------------- | ---------------- | --------------------------------------------- |
| 时间复杂度分析 | 分析循环嵌套层数 | O(1)常数、O(log n)对数、O(n)线性、O(n²)平方等 |
| 空间复杂度分析 | 计算额外内存使用 | 不包括输入数据占用的空间                      |
| 性能测试       | `clock()`函数    | 测量实际运行时间，需多次测试取平均值          |

**使用示例**

```c
#include <stdio.h>      // 标准输入输出
#include <time.h>       // 时间函数库
#include <stdlib.h>     // 标准库函数

// 线性查找函数，时间复杂度O(n)
int linearSearch(int arr[], int size, int target) {
    // 遍历整个数组查找目标值
    for (int i = 0; i < size; i++) {          // 循环n次，O(n)
        if (arr[i] == target) {               // 比较操作
            return i;                         // 找到返回索引
        }
    }
    return -1;                                // 未找到返回-1
}

// 计算函数执行时间的辅助函数
double measureTime(int (*func)(int[], int, int), int arr[], int size, int target) {
    clock_t start, end;                       // 声明开始和结束时间变量
    start = clock();                          // 记录开始时间
    
    int result = func(arr, size, target);     // 执行传入的函数
    
    end = clock();                            // 记录结束时间
    double cpu_time_used = ((double) (end - start)) / CLOCKS_PER_SEC;
                                              // 计算执行时间（秒）
    return cpu_time_used;                     // 返回执行时间
}

int main() {
    const int SIZE = 10000;                   // 定义数组大小
    int* arr = (int*)malloc(SIZE * sizeof(int)); // 动态分配数组内存
    
    // 初始化数组，填充0到9999
    for (int i = 0; i < SIZE; i++) {
        arr[i] = i;                           // 顺序填充
    }
    
    // 测试查找最后一个元素的时间
    double time_taken = measureTime(linearSearch, arr, SIZE, SIZE-1);
    printf("查找时间: %f 秒\n", time_taken);   // 输出执行时间
    
    free(arr);                                // 释放动态分配的内存
    return 0;
}
```

**注意事项**

- 大O表示法关注的是增长趋势，忽略常数因子
- 最坏情况、平均情况和最好情况复杂度可能不同
- 实际性能测试受硬件和系统负载影响，理论分析更重要

算法复杂度分析让我们能够预测程序在处理大规模数据时的表现。理解时间复杂度和空间复杂度是写出高效代码的关键。

### 1.3 C语言基础回顾

在深入数据结构之前，我们需要确保C语言基础扎实。毕竟，如果连指针都搞不清楚，那链表简直就是天书了！

**实例方法表格**

| 功能名称   | 实例调用方法                 | 具体功能与注意事项           |
| ---------- | ---------------------------- | ---------------------------- |
| 指针操作   | `int* ptr = &var;`           | 获取地址(&)和解引用(*)操作   |
| 动态内存   | `malloc/calloc/realloc/free` | 内存分配和释放，注意类型转换 |
| 数组与指针 | `arr[i] 等价于 *(arr+i)`     | 数组名可作为指针使用         |

**使用示例**

```c
#include <stdio.h>      // 标准输入输出
#include <stdlib.h>     // 动态内存分配函数

// 演示指针、数组和动态内存的基本操作
void demonstrateBasics() {
    // 基本变量和指针
    int value = 100;                          // 定义整型变量
    int* ptr = &value;                        // ptr指向value的地址
    
    printf("值: %d, 地址: %p\n", value, (void*)&value); // 输出值和地址
    printf("通过指针访问: %d\n", *ptr);        // 解引用指针获取值
    
    // 数组与指针的关系
    int arr[5] = {1, 2, 3, 4, 5};             // 定义并初始化数组
    int* arrPtr = arr;                        // 数组名自动转换为指针
    
    printf("arr[2] = %d, *(arr+2) = %d\n", arr[2], *(arr+2)); // 两种访问方式等价
    
    // 动态内存分配
    int size = 3;
    int* dynamicArr = (int*)malloc(size * sizeof(int)); // 分配3个整数的内存
    
    // 检查分配是否成功
    if (dynamicArr == NULL) {
        printf("内存分配失败！\n");
        return;
    }
    
    // 初始化动态数组
    for (int i = 0; i < size; i++) {
        dynamicArr[i] = i * 10;               // 赋值: 0, 10, 20
    }
    
    // 打印动态数组内容
    for (int i = 0; i < size; i++) {
        printf("dynamicArr[%d] = %d\n", i, dynamicArr[i]);
    }
    
    // 重新分配内存扩大数组
    int* resizedArr = (int*)realloc(dynamicArr, 5 * sizeof(int));
    if (resizedArr != NULL) {
        dynamicArr = resizedArr;              // 更新指针
        dynamicArr[3] = 30;                   // 新增元素
        dynamicArr[4] = 40;
        
        printf("扩容后第4个元素: %d\n", dynamicArr[3]);
    }
    
    free(dynamicArr);                         // 释放内存
}

int main() {
    demonstrateBasics();
    return 0;
}
```

**注意事项**

- 指针使用前必须初始化，野指针会导致程序崩溃
- malloc分配的内存内容是未初始化的，calloc会初始化为0
- realloc可能返回新的内存地址，原指针可能失效
- 每次malloc都要对应一次free，避免内存泄漏

扎实的C语言基础是学习数据结构的基石。特别是指针和内存管理，它们贯穿于所有复杂数据结构的实现中。
