# 第7章 二叉树

## 7.1 二叉树的存储

### 二叉树存储方式概述

二叉树作为树结构中最常用的形式，其存储方式直接影响到操作效率和内存使用。在C语言中，我们主要有两种存储方式：顺序存储和链式存储。选择哪种方式要根据具体的使用场景来决定。

**顺序存储 vs 链式存储对比表**

| 存储方式 | 内存效率                             | 操作便利性   | 适用场景       |
| -------- | ------------------------------------ | ------------ | -------------- |
| 顺序存储 | 完全二叉树时高效，普通二叉树浪费空间 | 索引计算简单 | 完全二叉树、堆 |
| 链式存储 | 按需分配，无浪费                     | 指针操作灵活 | 任意二叉树     |

### 顺序存储实现

顺序存储利用数组来存储二叉树节点，对于完全二叉树特别高效。在这种存储方式中，如果父节点在位置i，那么左子节点在2i+1，右子节点在2i+2。

```c
// 定义二叉树节点数据类型
#define MAX_SIZE 100
typedef char DataType;  // 节点数据类型，这里用字符作为示例

// 顺序存储的二叉树结构
typedef struct {
    DataType data[MAX_SIZE];  // 存储节点数据的数组
    int size;                 // 当前树中节点的数量
} SequentialBinaryTree;

// 初始化顺序二叉树
SequentialBinaryTree* initSequentialTree() {
    SequentialBinaryTree* tree = (SequentialBinaryTree*)malloc(sizeof(SequentialBinaryTree));
    if (tree == NULL) {
        printf("内存分配失败！\n");
        return NULL;
    }
    tree->size = 0;  // 初始大小为0
    return tree;
}

// 在指定位置插入节点
int insertNode(SequentialBinaryTree* tree, int index, DataType value) {
    // 检查参数有效性
    if (tree == NULL || index < 0 || index >= MAX_SIZE) {
        return -1;  // 返回-1表示错误
    }
    
    // 如果插入位置超出当前大小，需要填充空位置
    if (index >= tree->size) {
        // 填充中间的空位置（用特殊值表示空节点）
        for (int i = tree->size; i < index; i++) {
            tree->data[i] = '\0';  // 用空字符表示空节点
        }
        tree->size = index + 1;
    }
    
    tree->data[index] = value;  // 插入数据
    return 0;  // 返回0表示成功
}

// 获取父节点索引
int getParentIndex(int childIndex) {
    if (childIndex <= 0) {
        return -1;  // 根节点没有父节点
    }
    return (childIndex - 1) / 2;
}

// 获取左子节点索引
int getLeftChildIndex(int parentIndex) {
    return 2 * parentIndex + 1;
}

// 获取右子节点索引
int getRightChildIndex(int parentIndex) {
    return 2 * parentIndex + 2;
}

// 释放顺序二叉树内存
void freeSequentialTree(SequentialBinaryTree* tree) {
    if (tree != NULL) {
        free(tree);
    }
}
```

顺序存储的优点是访问速度快，通过简单的数学计算就能找到父子节点关系。但缺点也很明显：对于非完全二叉树会造成大量空间浪费，而且树的大小在创建时就需要确定。

### 链式存储实现

链式存储是二叉树最常用的存储方式，每个节点包含数据和指向左右子树的指针。

```c
// 二叉树节点结构定义
typedef struct TreeNode {
    DataType data;              // 节点数据
    struct TreeNode* left;      // 指向左子树的指针
    struct TreeNode* right;     // 指向右子树的指针
} TreeNode;

// 创建新节点
TreeNode* createNode(DataType data) {
    TreeNode* newNode = (TreeNode*)malloc(sizeof(TreeNode));
    if (newNode == NULL) {
        printf("内存分配失败！\n");
        return NULL;
    }
    newNode->data = data;       // 设置节点数据
    newNode->left = NULL;       // 左子树初始化为空
    newNode->right = NULL;      // 右子树初始化为空
    return newNode;
}

// 递归释放整个二叉树
void freeBinaryTree(TreeNode* root) {
    if (root == NULL) {
        return;  // 空树直接返回
    }
    freeBinaryTree(root->left);   // 递归释放左子树
    freeBinaryTree(root->right);  // 递归释放右子树
    free(root);                   // 释放当前节点
}

// 手动构建一个简单的二叉树用于测试
TreeNode* buildSampleTree() {
    // 创建根节点
    TreeNode* root = createNode('A');
    if (root == NULL) return NULL;
    
    // 创建左子树
    root->left = createNode('B');
    if (root->left == NULL) {
        freeBinaryTree(root);
        return NULL;
    }
    
    // 创建右子树
    root->right = createNode('C');
    if (root->right == NULL) {
        freeBinaryTree(root);
        return NULL;
    }
    
    // 继续构建更深层的节点
    root->left->left = createNode('D');
    root->left->right = createNode('E');
    root->right->left = createNode('F');
    
    return root;
}
```

链式存储的优势在于灵活性强，可以动态分配内存，适合任意形状的二叉树。每个节点只在需要时才分配内存，不会造成空间浪费。不过相比顺序存储，访问特定节点需要遍历，时间复杂度较高。

### 存储方式选择建议

选择存储方式时要考虑以下因素：

1. **树的形状**：如果是完全二叉树或接近完全二叉树，顺序存储更合适
2. **操作类型**：如果频繁需要随机访问节点，顺序存储更快
3. **内存限制**：内存紧张时，链式存储更节省空间
4. **插入删除频率**：频繁插入删除操作时，链式存储更灵活

对于大多数实际应用场景，链式存储是更好的选择，因为它适应性强，不会因为树的不平衡而浪费大量内存。

二叉树的存储是后续所有操作的基础，理解这两种存储方式的特点和适用场景，能帮助我们在实际开发中做出更好的技术选型。

## 7.2 二叉树的遍历

### 二叉树遍历基本概念

二叉树遍历是指按照某种规则访问二叉树中的所有节点，且每个节点仅被访问一次。遍历是二叉树最基本也是最重要的操作之一，很多其他操作都建立在遍历的基础上。

根据访问根节点的时机不同，遍历可以分为三种主要方式：

- **先序遍历（Pre-order）**：根 → 左 → 右
- **中序遍历（In-order）**：左 → 根 → 右  
- **后序遍历（Post-order）**：左 → 右 → 根

此外还有层序遍历（Level-order），按层次从上到下、从左到右访问节点。

### 先序遍历实现

先序遍历首先访问根节点，然后递归地先序遍历左子树，最后递归地先序遍历右子树。

```c
// 先序遍历 - 递归实现
void preorderTraversal(TreeNode* root) {
    if (root == NULL) {
        return;  // 空节点直接返回
    }
    
    printf("%c ", root->data);           // 访问根节点
    preorderTraversal(root->left);       // 递归遍历左子树
    preorderTraversal(root->right);      // 递归遍历右子树
}

// 先序遍历 - 非递归实现（使用栈）
void preorderTraversalIterative(TreeNode* root) {
    if (root == NULL) {
        return;
    }
    
    // 创建栈来模拟递归调用
    #define STACK_SIZE 100
    TreeNode* stack[STACK_SIZE];
    int top = -1;  // 栈顶指针
    
    TreeNode* current = root;
    
    while (current != NULL || top != -1) {
        // 一直向左走，访问每个节点并压入栈
        while (current != NULL) {
            printf("%c ", current->data);  // 访问当前节点
            
            // 将当前节点压入栈（为后续处理右子树做准备）
            if (top < STACK_SIZE - 1) {
                stack[++top] = current;
            } else {
                printf("栈溢出！\n");
                return;
            }
            
            current = current->left;  // 移动到左子节点
        }
        
        // 从栈中弹出节点，处理其右子树
        if (top != -1) {
            current = stack[top--];
            current = current->right;
        }
    }
}
```

先序遍历常用于复制二叉树、打印目录结构等场景，因为它是从根开始，符合我们通常的思维习惯。

### 中序遍历实现

中序遍历先递归遍历左子树，然后访问根节点，最后递归遍历右子树。对于二叉搜索树，中序遍历会得到有序序列。

```c
// 中序遍历 - 递归实现
void inorderTraversal(TreeNode* root) {
    if (root == NULL) {
        return;  // 空节点直接返回
    }
    
    inorderTraversal(root->left);        // 递归遍历左子树
    printf("%c ", root->data);           // 访问根节点
    inorderTraversal(root->right);       // 递归遍历右子树
}

// 中序遍历 - 非递归实现
void inorderTraversalIterative(TreeNode* root) {
    if (root == NULL) {
        return;
    }
    
    #define STACK_SIZE 100
    TreeNode* stack[STACK_SIZE];
    int top = -1;
    TreeNode* current = root;
    
    while (current != NULL || top != -1) {
        // 一直向左走到底，将路径上的所有节点压入栈
        while (current != NULL) {
            if (top < STACK_SIZE - 1) {
                stack[++top] = current;
            } else {
                printf("栈溢出！\n");
                return;
            }
            current = current->left;
        }
        
        // 弹出栈顶节点并访问
        if (top != -1) {
            current = stack[top--];
            printf("%c ", current->data);  // 访问节点
            
            // 转向右子树
            current = current->right;
        }
    }
}
```

中序遍历在二叉搜索树中特别有用，因为它能按升序输出所有元素，常用于排序和范围查询。

### 后序遍历实现

后序遍历先递归遍历左子树，再递归遍历右子树，最后访问根节点。这种遍历方式常用于释放内存、计算目录大小等需要先处理子节点的场景。

```c
// 后序遍历 - 递归实现
void postorderTraversal(TreeNode* root) {
    if (root == NULL) {
        return;  // 空节点直接返回
    }
    
    postorderTraversal(root->left);      // 递归遍历左子树
    postorderTraversal(root->right);     // 递归遍历右子树
    printf("%c ", root->data);           // 访问根节点
}

// 后序遍历 - 非递归实现（较复杂）
void postorderTraversalIterative(TreeNode* root) {
    if (root == NULL) {
        return;
    }
    
    #define STACK_SIZE 100
    TreeNode* stack[STACK_SIZE];
    int top = -1;
    TreeNode* lastVisited = NULL;  // 记录最后一个访问的节点
    TreeNode* current = root;
    
    while (current != NULL || top != -1) {
        if (current != NULL) {
            // 一直向左走，压入所有节点
            if (top < STACK_SIZE - 1) {
                stack[++top] = current;
            } else {
                printf("栈溢出！\n");
                return;
            }
            current = current->left;
        } else {
            // 查看栈顶节点
            TreeNode* peekNode = stack[top];
            
            // 如果右子树存在且未被访问过
            if (peekNode->right != NULL && lastVisited != peekNode->right) {
                current = peekNode->right;
            } else {
                // 访问栈顶节点
                printf("%c ", peekNode->data);
                lastVisited = stack[top--];
            }
        }
    }
}
```

后序遍历的非递归实现相对复杂，因为需要确保左右子树都被访问后才能访问根节点，所以需要额外的标记来记录访问状态。

### 层序遍历实现

层序遍历按层次从上到下、从左到右访问节点，通常使用队列来实现。

```c
// 层序遍历实现（使用队列）
void levelOrderTraversal(TreeNode* root) {
    if (root == NULL) {
        return;
    }
    
    #define QUEUE_SIZE 100
    TreeNode* queue[QUEUE_SIZE];
    int front = 0, rear = 0;
    
    // 根节点入队
    queue[rear++] = root;
    
    while (front != rear) {
        // 出队一个节点
        TreeNode* current = queue[front++];
        if (front >= QUEUE_SIZE) front = 0;  // 循环队列处理
        
        printf("%c ", current->data);  // 访问节点
        
        // 左子节点入队
        if (current->left != NULL) {
            queue[rear++] = current->left;
            if (rear >= QUEUE_SIZE) rear = 0;
        }
        
        // 右子节点入队
        if (current->right != NULL) {
            queue[rear++] = current->right;
            if (rear >= QUEUE_SIZE) rear = 0;
        }
    }
}
```

层序遍历在求树的高度、宽度，或者需要按层次处理节点时非常有用。

### 遍历方法对比总结

| 遍历方式 | 访问顺序 | 主要用途             | 递归难度 | 非递归难度 |
| -------- | -------- | -------------------- | -------- | ---------- |
| 先序遍历 | 根→左→右 | 复制树、表达式树求值 | 简单     | 中等       |
| 中序遍历 | 左→根→右 | 二叉搜索树排序       | 简单     | 中等       |
| 后序遍历 | 左→右→根 | 释放内存、计算大小   | 简单     | 困难       |
| 层序遍历 | 按层次   | 求高度、宽度         | 不适用   | 中等       |

掌握各种遍历方法的实现和应用场景，是理解和使用二叉树的关键。不同的遍历方式适用于不同的问题场景，选择合适的遍历方式能让代码更加简洁高效。

## 7.3 线索二叉树

### 线索二叉树基本概念

线索二叉树是一种对普通二叉树的改进，它利用二叉树中大量的空指针域来存储遍历序列中的前驱和后继信息。在n个节点的二叉树中，有n+1个空指针域，这些空间完全可以被利用起来。

线索化的基本思想是：

- 如果节点的左指针为空，则让它指向该节点在某种遍历序列中的前驱
- 如果节点的右指针为空，则让它指向该节点在某种遍历序列中的后继
- 为了区分指针是指向子节点还是线索，需要增加标志位

### 线索二叉树节点结构

首先我们需要重新定义二叉树节点结构，添加线索标志位。

```c
// 线索标志枚举
typedef enum {
    LINK,   // 指针指向子节点
    THREAD  // 指针指向线索（前驱或后继）
} PointerTag;

// 线索二叉树节点结构
typedef struct ThreadTreeNode {
    DataType data;                      // 节点数据
    struct ThreadTreeNode* left;        // 左指针
    struct ThreadTreeNode* right;       // 右指针
    PointerTag leftTag;                 // 左指针标志
    PointerTag rightTag;                // 右指针标志
} ThreadTreeNode;

// 创建线索二叉树节点
ThreadTreeNode* createThreadNode(DataType data) {
    ThreadTreeNode* node = (ThreadTreeNode*)malloc(sizeof(ThreadTreeNode));
    if (node == NULL) {
        printf("内存分配失败！\n");
        return NULL;
    }
    node->data = data;
    node->left = NULL;
    node->right = NULL;
    node->leftTag = LINK;    // 默认指向子节点
    node->rightTag = LINK;   // 默认指向子节点
    return node;
}
```

### 中序线索化实现

中序线索化是最常用的线索化方式，因为它在线索二叉树中应用最广泛。

```c
// 全局变量，用于记录访问过程中的前驱节点
ThreadTreeNode* pre = NULL;

// 中序线索化递归函数
void inThreading(ThreadTreeNode* root) {
    if (root == NULL) {
        return;
    }
    
    inThreading(root->left);  // 线索化左子树
    
    // 处理当前节点
    if (root->left == NULL) {
        root->left = pre;           // 左指针指向前驱
        root->leftTag = THREAD;     // 设置为线索
    }
    
    // 让前驱节点的右指针指向当前节点（如果是线索）
    if (pre != NULL && pre->right == NULL) {
        pre->right = root;          // 前驱的右指针指向当前节点
        pre->rightTag = THREAD;     // 设置为线索
    }
    
    pre = root;  // 更新前驱为当前节点
    
    inThreading(root->right);  // 线索化右子树
}

// 中序线索化主函数
ThreadTreeNode* inorderThreadedTree(ThreadTreeNode* root) {
    if (root == NULL) {
        return NULL;
    }
    
    pre = NULL;  // 初始化前驱为NULL
    inThreading(root);
    return root;
}
```

中序线索化的关键是维护一个全局的前驱指针，在遍历过程中不断更新前驱和当前节点之间的线索关系。

### 中序线索二叉树遍历

线索化完成后，我们可以不用递归或栈就能进行中序遍历，大大提高了空间效率。

```c
// 在中序线索二叉树中找到第一个节点（最左边的节点）
ThreadTreeNode* firstNode(ThreadTreeNode* root) {
    if (root == NULL) {
        return NULL;
    }
    
    ThreadTreeNode* current = root;
    // 一直向左走，直到找到最左边的节点
    while (current->leftTag == LINK && current->left != NULL) {
        current = current->left;
    }
    return current;
}

// 在中序线索二叉树中找到节点的后继
ThreadTreeNode* nextNode(ThreadTreeNode* node) {
    if (node == NULL) {
        return NULL;
    }
    
    // 如果右指针是线索，直接返回后继
    if (node->rightTag == THREAD) {
        return node->right;
    }
    
    // 如果右指针指向子节点，找右子树中最左边的节点
    ThreadTreeNode* current = node->right;
    while (current->leftTag == LINK && current->left != NULL) {
        current = current->left;
    }
    return current;
}

// 中序线索二叉树遍历（非递归，无需栈）
void inorderTraversalThreaded(ThreadTreeNode* root) {
    if (root == NULL) {
        return;
    }
    
    // 找到第一个节点
    ThreadTreeNode* current = firstNode(root);
    
    // 依次访问每个节点及其后继
    while (current != NULL) {
        printf("%c ", current->data);
        current = nextNode(current);
    }
}
```

线索二叉树遍历的优势在于不需要额外的栈空间，时间复杂度仍然是O(n)，但空间复杂度降到了O(1)。

### 构建和测试线索二叉树

下面是一个完整的示例，展示如何构建、线索化和遍历线索二叉树。

```c
// 释放线索二叉树内存（注意：不能直接free，因为有线索指针）
void freeThreadedTree(ThreadTreeNode* root) {
    // 对于线索二叉树，最安全的方式还是用普通后序遍历来释放
    // 这里简化处理，实际应用中需要小心处理线索指针
    if (root == NULL) return;
    
    // 临时保存子节点指针（因为线索可能修改了指针）
    ThreadTreeNode* left = (root->leftTag == LINK) ? root->left : NULL;
    ThreadTreeNode* right = (root->rightTag == LINK) ? root->right : NULL;
    
    freeThreadedTree(left);
    freeThreadedTree(right);
    free(root);
}

// 测试函数
void testThreadedTree() {
    // 构建一个简单的二叉树
    ThreadTreeNode* root = createThreadNode('A');
    root->left = createThreadNode('B');
    root->right = createThreadNode('C');
    root->left->left = createThreadNode('D');
    root->left->right = createThreadNode('E');
    
    printf("原始中序遍历: ");
    // 这里需要临时转换为普通二叉树进行遍历，或者实现普通遍历函数
    
    // 进行中序线索化
    inorderThreadedTree(root);
    
    printf("\n线索化后中序遍历: ");
    inorderTraversalThreaded(root);
    printf("\n");
    
    // 清理内存
    freeThreadedTree(root);
}
```

### 线索二叉树的优缺点

**优点：**

- 充分利用了空指针域，节省了存储空间
- 遍历时不需要栈或递归，空间复杂度为O(1)
- 可以快速找到节点的前驱和后继

**缺点：**

- 结构复杂，增加了编程难度
- 插入和删除操作比较复杂，需要维护线索关系
- 对于频繁修改的树结构不太适用

线索二叉树在需要频繁遍历且树结构相对稳定的场景下非常有用，比如编译器中的符号表、数据库索引等。虽然现代计算机内存充足，但在某些嵌入式系统或对性能要求极高的场景中，线索二叉树仍然是一个有价值的选择。

## 7.4 哈夫曼树

### 哈夫曼树基本概念

哈夫曼树（Huffman Tree）又称为最优二叉树，是一种带权路径长度最短的二叉树。它在数据压缩领域有着重要应用，特别是哈夫曼编码就是基于哈夫曼树实现的。

**基本概念：**

- **路径长度**：从树中一个节点到另一个节点之间的分支数
- **节点的带权路径长度**：该节点到树根之间的路径长度与节点权重的乘积
- **树的带权路径长度（WPL）**：树中所有叶子节点的带权路径长度之和

哈夫曼树的特点是权重越大的节点离根越近，这样可以最小化整体的带权路径长度。

### 哈夫曼树节点结构

哈夫曼树的节点需要存储权重信息，通常只关心叶子节点的数据。

```c
// 哈夫曼树节点结构
typedef struct HuffmanNode {
    int weight;                    // 节点权重
    char data;                     // 节点数据（仅叶子节点有意义）
    struct HuffmanNode* left;      // 左子树
    struct HuffmanNode* right;     // 右子树
} HuffmanNode;

// 创建哈夫曼树节点
HuffmanNode* createHuffmanNode(char data, int weight) {
    HuffmanNode* node = (HuffmanNode*)malloc(sizeof(HuffmanNode));
    if (node == NULL) {
        printf("内存分配失败！\n");
        return NULL;
    }
    node->data = data;
    node->weight = weight;
    node->left = NULL;
    node->right = NULL;
    return node;
}
```

### 最小堆实现（用于构建哈夫曼树）

构建哈夫曼树需要反复取出权重最小的两个节点，因此使用最小堆是最有效的方式。

```c
// 最小堆结构
typedef struct MinHeap {
    HuffmanNode** nodes;    // 节点指针数组
    int size;               // 当前堆大小
    int capacity;           // 堆容量
} MinHeap;

// 创建最小堆
MinHeap* createMinHeap(int capacity) {
    MinHeap* heap = (MinHeap*)malloc(sizeof(MinHeap));
    if (heap == NULL) {
        printf("内存分配失败！\n");
        return NULL;
    }
    heap->nodes = (HuffmanNode**)malloc(capacity * sizeof(HuffmanNode*));
    if (heap->nodes == NULL) {
        free(heap);
        printf("内存分配失败！\n");
        return NULL;
    }
    heap->size = 0;
    heap->capacity = capacity;
    return heap;
}

// 交换堆中两个节点
void swapNodes(HuffmanNode** a, HuffmanNode** b) {
    HuffmanNode* temp = *a;
    *a = *b;
    *b = temp;
}

// 最小堆向下调整
void minHeapify(MinHeap* heap, int idx) {
    int smallest = idx;
    int left = 2 * idx + 1;
    int right = 2 * idx + 2;
    
    // 找到三个节点中权重最小的
    if (left < heap->size && 
        heap->nodes[left]->weight < heap->nodes[smallest]->weight) {
        smallest = left;
    }
    
    if (right < heap->size && 
        heap->nodes[right]->weight < heap->nodes[smallest]->weight) {
        smallest = right;
    }
    
    // 如果最小的不是当前节点，交换并继续调整
    if (smallest != idx) {
        swapNodes(&heap->nodes[idx], &heap->nodes[smallest]);
        minHeapify(heap, smallest);
    }
}

// 插入节点到最小堆
void insertMinHeap(MinHeap* heap, HuffmanNode* node) {
    if (heap->size == heap->capacity) {
        printf("堆已满！\n");
        return;
    }
    
    // 插入到末尾
    heap->nodes[heap->size] = node;
    int i = heap->size;
    heap->size++;
    
    // 向上调整
    while (i != 0 && 
           heap->nodes[(i - 1) / 2]->weight > heap->nodes[i]->weight) {
        swapNodes(&heap->nodes[i], &heap->nodes[(i - 1) / 2]);
        i = (i - 1) / 2;
    }
}

// 提取最小权重节点
HuffmanNode* extractMin(MinHeap* heap) {
    if (heap->size <= 0) {
        return NULL;
    }
    if (heap->size == 1) {
        heap->size--;
        return heap->nodes[0];
    }
    
    // 取出根节点
    HuffmanNode* root = heap->nodes[0];
    
    // 将最后一个节点移到根位置
    heap->nodes[0] = heap->nodes[heap->size - 1];
    heap->size--;
    
    // 向下调整
    minHeapify(heap, 0);
    
    return root;
}

// 释放最小堆内存
void freeMinHeap(MinHeap* heap) {
    if (heap != NULL) {
        free(heap->nodes);
        free(heap);
    }
}
```

### 哈夫曼树构建算法

哈夫曼树的构建过程如下：

1. 将所有字符及其频率作为叶子节点放入最小堆
2. 重复以下步骤直到堆中只剩一个节点：
   - 取出权重最小的两个节点
   - 创建一个新节点，权重为两节点权重之和
   - 将这两个节点作为新节点的左右子树
   - 将新节点放回堆中
3. 堆中剩下的唯一节点就是哈夫曼树的根

```c
// 构建哈夫曼树
HuffmanNode* buildHuffmanTree(char data[], int freq[], int size) {
    if (size <= 0) {
        return NULL;
    }
    
    // 创建最小堆
    MinHeap* minHeap = createMinHeap(size);
    if (minHeap == NULL) {
        return NULL;
    }
    
    // 将所有字符节点插入堆中
    for (int i = 0; i < size; i++) {
        HuffmanNode* node = createHuffmanNode(data[i], freq[i]);
        if (node == NULL) {
            // 内存分配失败，清理已分配的内存
            for (int j = 0; j < i; j++) {
                free(minHeap->nodes[j]);
            }
            freeMinHeap(minHeap);
            return NULL;
        }
        insertMinHeap(minHeap, node);
    }
    
    // 构建哈夫曼树
    while (minHeap->size != 1) {
        // 取出两个最小权重的节点
        HuffmanNode* left = extractMin(minHeap);
        HuffmanNode* right = extractMin(minHeap);
        
        if (left == NULL || right == NULL) {
            // 错误处理
            freeHuffmanTree(left);
            freeHuffmanTree(right);
            freeMinHeap(minHeap);
            return NULL;
        }
        
        // 创建新节点
        HuffmanNode* newNode = createHuffmanNode('\0', 
                                               left->weight + right->weight);
        if (newNode == NULL) {
            freeHuffmanTree(left);
            freeHuffmanTree(right);
            freeMinHeap(minHeap);
            return NULL;
        }
        newNode->left = left;
        newNode->right = right;
        
        // 将新节点插入堆中
        insertMinHeap(minHeap, newNode);
    }
    
    // 返回哈夫曼树根节点
    HuffmanNode* root = extractMin(minHeap);
    freeMinHeap(minHeap);
    return root;
}
```

### 哈夫曼编码生成

有了哈夫曼树，我们就可以生成每个字符的哈夫曼编码。

```c
// 生成哈夫曼编码的辅助函数
void generateHuffmanCodes(HuffmanNode* root, char* code, int top, 
                         char result[][256], char data[]) {
    if (root == NULL) {
        return;
    }
    
    // 如果是叶子节点，保存编码
    if (root->left == NULL && root->right == NULL) {
        code[top] = '\0';
        // 找到对应的字符并保存编码
        for (int i = 0; data[i] != '\0'; i++) {
            if (data[i] == root->data) {
                strcpy(result[i], code);
                break;
            }
        }
        return;
    }
    
    // 向左走，编码加'0'
    code[top] = '0';
    generateHuffmanCodes(root->left, code, top + 1, result, data);
    
    // 向右走，编码加'1'
    code[top] = '1';
    generateHuffmanCodes(root->right, code, top + 1, result, data);
}

// 主函数：生成所有字符的哈夫曼编码
void printHuffmanCodes(HuffmanNode* root, char data[], int size) {
    if (root == NULL) {
        return;
    }
    
    // 为每个字符的编码分配空间
    char codes[size][256];
    char tempCode[256];
    
    // 生成编码
    generateHuffmanCodes(root, tempCode, 0, codes, data);
    
    // 打印结果
    printf("字符\t频率\t哈夫曼编码\n");
    for (int i = 0; i < size; i++) {
        printf("%c\t%d\t%s\n", data[i], /* 需要传入频率数组 */, codes[i]);
    }
}

// 释放哈夫曼树内存
void freeHuffmanTree(HuffmanNode* root) {
    if (root == NULL) {
        return;
    }
    freeHuffmanTree(root->left);
    freeHuffmanTree(root->right);
    free(root);
}
```

### 哈夫曼树的应用价值

哈夫曼树的主要应用包括：

1. **数据压缩**：哈夫曼编码是最经典的无损压缩算法之一
2. **文件压缩**：ZIP、GZIP等压缩工具都使用了类似的思想
3. **网络传输**：减少传输数据量，提高传输效率
4. **存储优化**：节省磁盘空间

哈夫曼树的优势在于它能根据字符出现的频率自适应地生成最优编码，频率高的字符用短编码，频率低的字符用长编码，从而达到整体最优的压缩效果。

需要注意的是，哈夫曼编码需要传输编码表（或者哈夫曼树本身），这会增加一些额外开销，但对于大文件来说，这个开销通常可以忽略不计。
