### 15.1 排序基本概念

排序是数据处理中非常基础且重要的操作，简单来说就是将一组无序的数据按照特定规则（通常是升序或降序）重新排列。在实际开发中，无论是数据库查询结果的展示，还是算法优化中的预处理步骤，排序都扮演着不可或缺的角色。

| 技术点       | 说明                           |
| ------------ | ------------------------------ |
| 当前讲解内容 | 排序的基本概念和分类           |
| 适用场景     | 数据整理、搜索优化、结果展示等 |
| 前置知识要求 | C语言基础、数组操作            |

**排序的分类方式：**

- **内部排序 vs 外部排序**：内部排序指所有数据都能加载到内存中进行排序；外部排序则用于数据量太大无法全部载入内存的情况。
- **稳定 vs 不稳定**：稳定排序保证相等元素的相对位置不变，比如冒泡排序；不稳定排序如快速排序则不保证这一点。
- **比较类 vs 非比较类**：比较类排序通过元素间的比较决定顺序（如快速排序），非比较类排序则利用数据特性直接定位（如计数排序）。

这节主要介绍了排序的基本概念及其常见分类方式，帮助我们理解不同排序算法的应用场景和特点。

### 15.2 稳定性分析

稳定性是评价排序算法的一个重要指标。所谓稳定排序，指的是在排序过程中，相等元素的相对位置不会发生变化。举个例子，如果有一组学生按成绩排序，使用稳定排序后，相同成绩的学生仍然保持原来的录入顺序。

| 功能名称   | 实例调用方法     | 具体功能与注意事项                                         |
| ---------- | ---------------- | ---------------------------------------------------------- |
| 判断稳定性 | 人工分析算法逻辑 | 需要检查算法是否在交换或移动元素时破坏了相等元素的原始顺序 |
| 稳定性测试 | 编写测试用例验证 | 构造包含重复元素的测试数据，观察排序前后相等元素的位置变化 |

```c
// 示例：验证冒泡排序的稳定性
#include <stdio.h>
#include <stdbool.h>

// 定义结构体存储值和原始索引
typedef struct {
    int value;
    int originalIndex;
} Element;

// 冒泡排序实现（稳定）
void bubbleSort(Element arr[], int n) {
    // 外层循环控制排序轮数
    for (int i = 0; i < n - 1; i++) {
        // 内层循环进行相邻元素比较
        for (int j = 0; j < n - i - 1; j++) {
            // 只有当前元素大于后一个元素时才交换（注意这里没有等于的情况）
            if (arr[j].value > arr[j + 1].value) {
                // 交换两个元素
                Element temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}

// 测试稳定性
bool isStable(Element original[], Element sorted[], int n) {
    // 检查所有相等元素的原始索引是否保持递增
    for (int i = 0; i < n; i++) {
        for (int j = i + 1; j < n; j++) {
            if (original[i].value == original[j].value) {
                // 找到排序后的对应位置
                int idx1 = -1, idx2 = -1;
                for (int k = 0; k < n; k++) {
                    if (sorted[k].value == original[i].value && 
                        sorted[k].originalIndex == original[i].originalIndex) {
                        idx1 = k;
                    }
                    if (sorted[k].value == original[j].value && 
                        sorted[k].originalIndex == original[j].originalIndex) {
                        idx2 = k;
                    }
                }
                // 如果原始索引小的元素在排序后位置更大，则不稳定
                if (idx1 > idx2) {
                    return false;
                }
            }
        }
    }
    return true;
}

int main() {
    // 创建测试数据
    Element data[] = {{3, 0}, {1, 1}, {3, 2}, {2, 3}};
    int n = sizeof(data) / sizeof(data[0]);
    
    // 保存原始数据用于比较
    Element original[4];
    for (int i = 0; i < n; i++) {
        original[i] = data[i];
    }
    
    // 执行排序
    bubbleSort(data, n);
    
    // 验证稳定性
    if (isStable(original, data, n)) {
        printf("冒泡排序是稳定的\n");
    } else {
        printf("冒泡排序不稳定\n");
    }
    
    return 0;
}
```

**注意事项：**

- 稳定性只对包含重复元素的数据有意义
- 在实现排序算法时，比较条件中避免使用 >= 或 <=，而应该使用 > 或 < 来保持稳定性
- 稳定性对于某些应用场景非常重要，比如多级排序（先按姓名排序，再按年龄排序）

这节详细解释了排序稳定性的概念，并通过冒泡排序的例子展示了如何验证和保持排序的稳定性。

### 15.3 复杂度对比

时间复杂度和空间复杂度是衡量排序算法效率的核心指标。不同的排序算法在不同场景下表现差异很大，了解这些差异有助于我们在实际开发中选择最合适的算法。

| 排序算法 | 最好时间复杂度 | 平均时间复杂度 | 最坏时间复杂度 | 空间复杂度 | 稳定性 |
| -------- | -------------- | -------------- | -------------- | ---------- | ------ |
| 冒泡排序 | O(n)           | O(n²)          | O(n²)          | O(1)       | 稳定   |
| 选择排序 | O(n²)          | O(n²)          | O(n²)          | O(1)       | 不稳定 |
| 插入排序 | O(n)           | O(n²)          | O(n²)          | O(1)       | 稳定   |
| 快速排序 | O(n log n)     | O(n log n)     | O(n²)          | O(log n)   | 不稳定 |
| 归并排序 | O(n log n)     | O(n log n)     | O(n log n)     | O(n)       | 稳定   |
| 堆排序   | O(n log n)     | O(n log n)     | O(n log n)     | O(1)       | 不稳定 |

**复杂度分析要点：**

- **时间复杂度**：反映算法执行时间随数据规模增长的变化趋势
- **空间复杂度**：反映算法额外内存消耗随数据规模增长的变化趋势  
- **最好/最坏情况**：某些算法在特定数据分布下表现优异（如插入排序对近似有序数据）

```c
// 示例：性能测试框架（简化版）
#include <stdio.h>
#include <time.h>
#include <stdlib.h>

// 生成随机数组
void generateRandomArray(int arr[], int n) {
    // 使用当前时间作为随机种子
    srand((unsigned int)time(NULL));
    // 填充随机数
    for (int i = 0; i < n; i++) {
        arr[i] = rand() % 1000; // 生成0-999的随机数
    }
}

// 冒泡排序实现
void bubbleSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}

// 插入排序实现
void insertionSort(int arr[], int n) {
    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int j = i - 1;
        // 将大于key的元素向后移动
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}

// 性能测试函数
double measureTime(void (*sortFunc)(int[], int), int arr[], int n) {
    // 复制数组避免修改原数据
    int *testArr = (int*)malloc(n * sizeof(int));
    for (int i = 0; i < n; i++) {
        testArr[i] = arr[i];
    }
    
    // 记录开始时间
    clock_t start = clock();
    // 执行排序
    sortFunc(testArr, n);
    // 记录结束时间
    clock_t end = clock();
    
    // 释放内存
    free(testArr);
    
    // 返回执行时间（秒）
    return ((double)(end - start)) / CLOCKS_PER_SEC;
}

int main() {
    const int sizes[] = {1000, 5000, 10000};
    const int numSizes = sizeof(sizes) / sizeof(sizes[0]);
    
    printf("排序算法性能对比（时间单位：秒）\n");
    printf("数据规模\t冒泡排序\t插入排序\n");
    
    for (int i = 0; i < numSizes; i++) {
        int n = sizes[i];
        int *arr = (int*)malloc(n * sizeof(int));
        
        // 生成随机测试数据
        generateRandomArray(arr, n);
        
        // 测试冒泡排序
        double bubbleTime = measureTime(bubbleSort, arr, n);
        // 测试插入排序
        double insertionTime = measureTime(insertionSort, arr, n);
        
        printf("%d\t\t%.4f\t\t%.4f\n", n, bubbleTime, insertionTime);
        
        free(arr);
    }
    
    return 0;
}
```

**注意事项：**

- 实际性能测试需要多次运行取平均值以减少误差
- 不同硬件环境下的测试结果会有差异
- 对于小规模数据，算法常数因子的影响可能超过复杂度本身
- 内存访问模式也会影响实际性能（缓存友好性）

这节通过表格对比了常见排序算法的复杂度特征，并提供了简单的性能测试框架，帮助我们理解理论复杂度与实际性能的关系。
